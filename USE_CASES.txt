Authentication & User Management
- Register account: Create new user (farmer or enterprise).
  Endpoint: POST /api/v1/auth/register
  Access: Public
  Request: { fullName, email, phone, password, confirmPassword, role, agreeTerms }

- Login: Authenticate user and receive access + refresh tokens.
  Endpoint: POST /api/v1/auth/login
  Access: Public
  Request: { emailOrPhone, password, rememberMe }

- Refresh access token: Exchange refresh token for new access token.
  Endpoint: POST /api/v1/auth/refresh-token
  Access: Public (requires refresh token cookie or body)

- Logout: Invalidate refresh token (clears cookie).
  Endpoint: POST /api/v1/auth/logout
  Access: Private

- Forgot password: Request password reset token (email flow).
  Endpoint: POST /api/v1/auth/forgot-password
  Access: Public
  Request: { email }

- Reset password: Reset using token sent via email.
  Endpoint: POST /api/v1/auth/reset-password
  Access: Public
  Request: { token, password, confirmPassword }

- Update password: Change password while authenticated.
  Endpoint: PUT /api/v1/auth/update-password
  Access: Private
  Request: { currentPassword, newPassword, confirmNewPassword }

- Get current user (profile): Retrieve logged-in user's details.
  Endpoint: GET /api/v1/auth/me
  Access: Private

- Update profile: Update name/phone/avatar (not password).
  Endpoint: PUT /api/v1/auth/update-profile
  Access: Private
  Request: { fullName, phone, avatar }

- Verify email: Verify account by token link.
  Endpoint: GET /api/v1/auth/verify-email/:token
  Access: Public

- Resend verification email: Trigger sending verification link.
  Endpoint: POST /api/v1/auth/resend-verification
  Access: Private

- Deactivate account: Soft-delete user account.
  Endpoint: DELETE /api/v1/auth/deactivate
  Access: Private

Products & Reviews
- List products: Query/filter/sort/paginate products.
  Endpoint: GET /api/v1/products
  Access: Public
  Query: category, region, type, search, page, limit, sort

- Get product by ID: Retrieve single product details.
  Endpoint: GET /api/v1/products/:id
  Access: Public

- Get similar products: Recommendations by product.
  Endpoint: GET /api/v1/products/:id/similar
  Access: Public

- Get products by region: region-based listing (north/central/south).
  Endpoint: GET /api/v1/products/region/:region
  Access: Public

- Create product listing: Farmer creates a product/crop.
  Endpoint: POST /api/v1/products
  Access: Private (Farmer only)
  Request: product data

- Update product: Owner updates product details.
  Endpoint: PUT /api/v1/products/:id
  Access: Private (Owner)

- Delete product: Soft-delete product.
  Endpoint: DELETE /api/v1/products/:id
  Access: Private (Owner)

- Get my products: List products created by current user.
  Endpoint: GET /api/v1/products/my
  Access: Private

- Get product reviews: List reviews for a product.
  Endpoint: GET /api/v1/products/:id/reviews
  Access: Public

- Add product review: Enterprise can add review to a product.
  Endpoint: POST /api/v1/products/:id/reviews
  Access: Private (Enterprise only)
  Request: { rating, text }

Escrow & Dispute Flow
- Create escrow: Initialize escrow for a contract.
  Endpoint: POST /api/v1/escrow
  Access: Private
  Request: { contractId }

- List escrows: List escrows for user.
  Endpoint: GET /api/v1/escrow
  Access: Private

- Get escrow by ID: Details and milestones.
  Endpoint: GET /api/v1/escrow/:id
  Access: Private

- Get escrow by contract: Fetch escrow by contract ID.
  Endpoint: GET /api/v1/escrow/contract/:contractId
  Access: Private

- Deposit: Enterprise deposits funds into escrow.
  Endpoint: POST /api/v1/escrow/:id/deposit
  Access: Private (Enterprise)
  Request: { amount }

- Farmer confirm milestone: Farmer confirms a milestone (evidence allowed).
  Endpoint: POST /api/v1/escrow/:id/farmer-confirm
  Access: Private (Farmer)
  Request: { milestoneStep, evidence }

- Enterprise confirm milestone: Enterprise confirms milestone.
  Endpoint: POST /api/v1/escrow/:id/enterprise-confirm
  Access: Private (Enterprise)
  Request: { milestoneStep, evidence }

- Raise dispute: Create dispute for a milestone.
  Endpoint: POST /api/v1/escrow/:id/dispute
  Access: Private
  Request: { milestoneStep, reason, evidence[] }

- List user disputes: Get disputes belonging to user.
  Endpoint: GET /api/v1/escrow/disputes
  Access: Private

- Resolve dispute: Admin (currently any auth) resolves dispute.
  Endpoint: POST /api/v1/escrow/disputes/:id/resolve
  Access: Private (should be Admin in production)
  Request: { resolution, adminNotes }

- Get virtual balance: Retrieve user's virtual wallet balance.
  Endpoint: GET /api/v1/escrow/balance
  Access: Private

Notes & Observations
- File-based uploads (`/uploads`) are stored on local filesystem — not suitable for ephemeral hosting.
- Payment webhook endpoint is public — requires secure verification of gateway signatures.
- Some admin-only operations (dispute resolution) are currently not restricted — should be enforced in production.
